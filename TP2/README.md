ALGO 1
Dans cet algorithme, le tableau est parcouru une seule fois. À chaque étape, on vérifie si l’opposé de l’élément courant est déjà présent dans l’ensemble vus. Comme la recherche et l’ajout dans un ensemble se font en moyenne très rapidement, la complexité en temps est en O(n).


ALGO 2

Dans cet algorithme, trois boucles sont utilisées pour tester toutes les combinaisons possibles de trois éléments du tableau. Dans le pire des cas, il faut donc parcourir un très grand nombre de triplets, ce qui donne une complexité en temps en O(n³).
